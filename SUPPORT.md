# Support

## Where to ask

- Use GitHub Issues for reproducible bugs and narrowly scoped feature requests.
- Use GitHub Discussions, if enabled, for configuration questions, design
  discussion, and sharing deployment experience.
- Use [SECURITY.md](SECURITY.md) for suspected vulnerabilities. Do not post
  security-sensitive details publicly.

## What to include

Redact all private data. Useful details include the module or commit version,
Terraform and AWS provider versions, Kroxylicious image digest, a minimal
configuration that omits secrets, and the exact error message.

Never share AWS access keys, SASL/SCRAM credentials, TLS private keys,
unredacted Terraform state, or internal broker hostnames.

## Support expectations

This is an experimental open-source project maintained on a best-effort basis.
No response-time or production-support commitment is implied.
